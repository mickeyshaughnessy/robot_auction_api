Robot Services Exchange (RSX) FAQ

General Information

What is the Robot Services Exchange (RSX)?
The Robot Services Exchange (RSX) is a platform designed to facilitate the buying and selling of robot services. It acts as an intermediary between robot owners (sellers) and people who need robot services (buyers). Think of it as a job board for robots, where tasks are posted and robots are assigned to complete them.
How does RSX work?
RSX works through a bidding system:
  1. Buyers submit bids for specific services they need, including details like the type of service, location, price, and time frame.
  2. Robot owners (sellers) can browse available jobs and accept those that match their robots' capabilities and location.
  3. When a match is made, the buyer pays for the service, the robot performs the task, and then the seller is paid.
This system allows for dynamic pricing and efficient allocation of robot resources based on supply and demand.

For Buyers

How do I place a bid for a robot service?
To place a bid, you need to use the `/make_bid` endpoint of our API. Here's a step-by-step process:
  1. Ensure you have registered an account and obtained an authentication token.
  2. Prepare your bid data, including:
    • service: The type of service you need (e.g., cleaning, delivery)
    • lat: Latitude of the service location
    • lon: Longitude of the service location
    • price: Your bid price for the service (in USD)
    • end_time: The deadline for the service (in Unix timestamp format)
  3. Send a POST request to `https://100.26.236.1:5001/make_bid` with your bid data in JSON format, including your authentication token in the header.
The system will then process your bid and make it available for robot owners to accept.

Technical Information

What technologies does RSX use?
RSX is built using a variety of modern technologies:
  • Backend: The core API is built with Python using the Flask framework.
  • Database: We use Redis for fast, in-memory data storage and retrieval.
  • Server: The application is served using Gunicorn, a Python WSGI HTTP Server.
  • Frontend: The dashboard uses HTML, CSS, and JavaScript with the Leaflet library for mapping.
  • API Communication: RESTful API endpoints are used for all client-server communication.
  • Authentication: We use JWT (JSON Web Tokens) for secure user authentication.
This tech stack allows for a scalable, efficient, and secure platform for robot service exchange.
