Robot Services Exchange (RSX) FAQ

General Information

Q: What is the Robot Services Exchange (RSX)?
A: The Robot Services Exchange (RSX) is a platform designed to facilitate the buying and selling of robot services. It acts as an intermediary between robot owners (sellers) and people who need robot services (buyers). Think of it as a job board for robots, where tasks are posted and robots are assigned to complete them.
Q: How does RSX work?
A: RSX works through a bidding system:
  1. Buyers submit bids for specific services they need, including details like the type of service, location, price, and time frame.
  2. Robot owners (sellers) can browse available jobs and accept those that match their robots' capabilities and location.
  3. When a match is made, the buyer pays for the service, the robot performs the task, and then the seller is paid.
This system allows for dynamic pricing and efficient allocation of robot resources based on supply and demand.

For Buyers

Q: How do I place a bid for a robot service?
A: To place a bid, you need to use the `/make_bid` endpoint of our API. Here's a step-by-step process:
  1. Ensure you have registered an account and obtained an authentication token.
  2. Prepare your bid data, including:
    • service: The type of service you need (e.g., cleaning, delivery)
    • lat: Latitude of the service location
    • lon: Longitude of the service location
    • price: Your bid price for the service (in USD)
    • end_time: The deadline for the service (in Unix timestamp format)
  3. Send a POST request to `https://100.26.236.1:5001/make_bid` with your bid data in JSON format, including your authentication token in the header.
The system will then process your bid and make it available for robot owners to accept.
Q: How do I know if my bid has been accepted?
A: Currently, there isn't a direct notification system in place. However, you can check the status of your bid by using the `/nearby` endpoint. This will show you recent activities in your area, including accepted bids. In the future, we plan to implement a more robust tracking system for individual bids.

For Sellers (Robot Owners)

Q: How do I register my robot for services?
A: Robot registration is handled through our client application. Here's how you can do it:
  1. First, ensure you have the latest version of our RobotOwnerClient software.
  2. Use the `register_robot` method to add your robot to the system. You'll need to provide:
    • robot_id: A unique identifier for your robot
    • services: A list of services your robot can perform
    • lat: The current latitude of your robot
    • lon: The current longitude of your robot
    • max_distance: The maximum distance your robot is willing to travel for a job
  3. Once registered, your robot will be eligible to receive job offers in its area.
Remember to keep your robot's location updated if it moves to ensure it receives relevant job offers.
Q: How does my robot get assigned to jobs?
A: Job assignment in RSX is proactive on the seller's side. Here's how it works:
  1. Our system continuously checks for available jobs that match your robot's capabilities and location.
  2. When a matching job is found, it's offered to your robot through the `/grab_job` endpoint.
  3. If your robot accepts the job (which is automated in the RobotOwnerClient), it's assigned to complete the task.
  4. After completing the job, make sure to update the job status using the `/sign_job` endpoint.
This system ensures that your robot can quickly find and accept suitable jobs in its area.

Technical Information

Q: What technologies does RSX use?
A: RSX is built using a variety of modern technologies:
  • Backend: The core API is built with Python using the Flask framework.
  • Database: We use Redis for fast, in-memory data storage and retrieval.
  • Server: The application is served using Gunicorn, a Python WSGI HTTP Server.
  • Frontend: The dashboard uses HTML, CSS, and JavaScript with the Leaflet library for mapping.
  • API Communication: RESTful API endpoints are used for all client-server communication.
  • Authentication: We use JWT (JSON Web Tokens) for secure user authentication.
This tech stack allows for a scalable, efficient, and secure platform for robot service exchange.
Q: Is there an API documentation available?
A: Yes, we have comprehensive API documentation available. You can find detailed information about all our endpoints, including required parameters and response formats, in our `index.html` file. This documentation covers:
  • Authentication endpoints (/register, /login)
  • Account management endpoints (/account_data)
  • Buyer endpoints (/make_bid)
  • Seller endpoints (/grab_job)
  • Shared endpoints (/nearby, /sign_job)
For the most up-to-date information, always refer to the official documentation or contact our support team.

Security and Privacy

Q: How secure is the RSX platform?
A: Security is a top priority for RSX. We've implemented several measures to ensure the safety of our users and their data:
  1. Authentication: We use secure token-based authentication for all API requests.
  2. Password Security: User passwords are hashed before storage, ensuring that even in the unlikely event of a data breach, passwords remain secure.
  3. HTTPS: All communication with our API is encrypted using HTTPS.
  4. Rate Limiting: We implement rate limiting to prevent abuse of our API.
  5. Regular Audits: Our system undergoes regular security audits to identify and address potential vulnerabilities.
However, it's important to note that no system is 100% secure. We continuously work on improving our security measures and encourage users to follow best practices, such as using strong, unique passwords and not sharing their authentication tokens.
Q: How is my personal data handled?
A: We take data privacy very seriously. Here's how we handle personal data:
  1. Data Collection: We only collect information that's necessary for the functioning of the platform, such as usernames, encrypted passwords, and service-related data.
  2. Data Usage: Your data is used solely for the purpose of facilitating robot services and improving our platform.
  3. Data Storage: We use secure, encrypted databases to store user information.
  4. Data Sharing: We do not sell or share your personal data with third parties.
  5. Data Access: You can view your account data using the `/account_data` endpoint.
  6. Data Deletion: If you wish to delete your account and associated data, please contact our support team.
For more detailed information, please refer to our privacy policy.
